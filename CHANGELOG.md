# 更新日志

文档导航：[A0 文档索引](./docs/A0_DOC_INDEX.md)

## v1.1.0

本版本重点优化 PDF 来源定位和重复查询性能。

发布候选更新日期：2026-07-14

- 发布前安全与一致性复核补齐文档ID路径边界、缓存按需增强、分析运行上下文绑定、跨claim计算、下载来源身份、SQLite索引一致性和表格失败清理；发布包测试数量以`docs/C1_TEST_PLAN.md`中的最终复核记录为准。
- 移除未被实现使用的Camelot、OpenCV、ChromaDB和sentence-transformers依赖；表格抽取只声明实际使用的pdfplumber，向量接口明确为外部后端清单，不再把占位文件描述为已生成embedding索引。

- 最终发布候选通过294项自动化测试、Ruff、Mypy、Python编译和sdist/wheel构建验证。
- A股公告、年报、招股书和融资文件查询的默认结束日期改为系统当前日期，消除固定`20261231`导致2027年后默认漏查新公告的问题。
- checkpoint新增输入文件SHA-256、有效运行参数和schema版本校验；同路径输入内容或OCR等关键参数变化时拒绝错误复用旧结果。
- PDF下载重试覆盖HTTP 408/425/429/5xx和流式传输中断，支持`Retry-After`，并使用按目标文件加锁及唯一临时文件避免并发碰撞；MD5和SHA-256在下载写入时同步计算，不再额外读取整份文件。
- SQLite连接改为显式提交、回滚和关闭，修复Windows下数据库文件使用后仍可能被锁定的问题；schema按数据库路径只初始化一次，页面与FTS写入改为批量执行。
- 新增SQLite trigram索引加速中文子串检索，现有数据库自动迁移到schema v3；3000页本地性能冒烟中检索约20毫秒。
- 批量进度回调改为维护增量完成列表，5000项无I/O性能冒烟约15毫秒，不再每完成一项重新排序并复制全部结果。
- HKEX PDF存在性检查在HEAD不可用时回退到Range GET和PDF文件头校验，减少有效中文配对文件被误判不存在。
- GitHub Actions拆分质量检查、跨平台测试矩阵、完整extras冒烟和单次发布构建，避免在8个矩阵任务中重复运行Ruff、Mypy及构建。
- `--stop-on-error`在未处理完全部任务时保留checkpoint，后续`--resume`可继续剩余任务；完整批次仍自动删除checkpoint。
- SQLite全文索引写入改为进程内串行、读取及下载解析继续并行，消除大文件并发ingest偶发的`database is locked`。
- 港股年报完整性校验兼容美式`Notes to Consolidated Financial Statements`标题，不再误要求必须包含`the`；万国数据374页2025年报实测通过。
- 中文长问题的子串兜底先查整句，再按最多16个关键词召回并按关键词覆盖度排序；财务分析为用户原始问题保留两个优先证据位，绿联科技和达梦数据招股书可直接返回收入金额表。
- 第6轮12份真实文件从空数据目录完成下载、结构及身份校验和ingest；首次批次在约144.15秒完成11份并暴露万国数据标题兼容问题，修复后单项约16.94秒通过，最终全清单热缓存复跑内部耗时约0.88秒且`12/12`命中缓存。
- 批量CLI新增`--summary-only`；指定`--output`时完整结果继续写入JSON文件，终端可只显示总体统计和逐项状态，减少大批量校验明细占用上下文。
- 基础安装拆出`company-data`和`mcp`可选依赖；轻量核心不再强制安装AKShare、pandas、openpyxl和MCP，一键安装仍默认提供`pdf,company-data,mcp`完整常用能力。
- 增强Markdown版面转换单独拆为`layout`可选依赖；常规PDF ingest不再安装PyMuPDF4LLM及其版面模型依赖，降低默认安装体积和耗时。
- 包许可证元数据改用标准SPDX表达式，消除新版setuptools的弃用警告。
- `dev`可选依赖补齐测试直接使用的Pillow和pytesseract，并移除测试对pandas的非必要依赖，确保拆分运行依赖后GitHub Actions仍可在纯净环境执行完整测试。
- `requirements.txt`改为引用`pyproject.toml`中的常用extras，避免旧依赖清单重新安装已拆出的版面模型或继续发生漂移。
- wheel元数据补齐GitHub主页、仓库、Issue和更新日志链接；README与环境变量示例改为正式发布口径。
- 批量完整结果JSON现在包含自身`output_path`，与命令返回值保持一致。
- GitHub Actions新增版本tag与手工触发、最小只读权限、任务超时，以及独立的sdist和wheel构建；分步构建同时避开Windows Python 3.14嵌套隔离环境异常。
- CNINFO查询、详情页和PDF下载端点统一由明文HTTP切换为HTTPS，避免公开披露文件在传输途中被篡改。
- 轻量核心移除未直接使用的httpx和lxml；MCP与AKShare安装时仍由对应extras带入所需版本，基础来源查询安装更快。
- 默认缓存审计先按文件大小筛选重复候选，只对同大小文件计算SHA-256；完整正文审计仍全量哈希，重复检测准确性不变。
- 修复wheel安装后默认数据目录落入Python `Lib`目录的问题；源码工作区继续使用项目数据目录，wheel改用操作系统用户数据目录，`AH_DISCLOSURE_DATA_DIR`仍具有最高优先级。
- 批量任务增加完全重复输入复用和规范化文件任务锁，避免重复行或证券代码别名并发写入同一文件；`effective_workers`改为报告真实线程数。
- 下载连接使用10秒连接超时、60秒读取超时和最多2次尝试，降低失效链接占用批量线程的长尾等待。
- CLI服务信息改用轻量模块，不再为普通CLI命令提前加载完整MCP运行时。
- A股融资文件多分类搜索在全部来源失败时返回结构化错误，不再把来源异常静默表现为无结果。
- 大批量checkpoint按任务规模限频原子写入，完成结果及断点续跑语义保持不变。
- 新增Windows/Linux、Python 3.11/3.12/3.13/3.14 GitHub Actions测试矩阵，并由独立任务执行Ruff、Mypy、完整extras冒烟和发布包构建。
- 内容审计在PDF SHA-256与索引页数一致时复用`pages.jsonl`，仅对缓存缺失或不一致文件重新扫描PDF，并分别报告复用和扫描数量。
